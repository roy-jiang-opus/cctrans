**Двуязычный оверлей** для Claude Code: строка перевода (китайский / японский / корейский / русский / хинди / испанский / португальский / французский / немецкий) под каждой английской строкой, **прямо в диалоге** — только для отображения, поэтому транскрипт, контекст модели и ваш счёт за токены остаются на 100% английскими.

## ✨ Возможности

- 🪞 **Встроенное двуязычное отображение** — перевод появляется под каждой английской строкой по мере стриминга ответа
- 🧩 **Три варианта компоновки** — построчное чередование, по блокам (`cctrans mode section`) или весь ответ целиком (`cctrans mode message`)
- 🔄 **Под текстом или вместо него** — показывать перевод под английским или, через `cctrans display replace`, показывать только перевод на его месте
- ❓ **Перевод диалогов с вопросами** — интерактивные диалоги-вопросы Claude Code тоже показываются на вашем языке, а модель по-прежнему читает ваш ответ на английском
- 🧾 **Неразрушающий** — транскрипт и контекст модели остаются чисто английскими; skills, документация и код не затронуты
- 🆓 **Ноль токенов основного цикла** — перевод идёт через отдельный дешёвый (или бесплатный) бэкенд, полностью вне сессии Claude Code
- ⌨️ **Перевод ввода (beta)** — печатайте на родном языке, модель работает — и отвечает — на английском (`cctrans input on`)
- 🌏 **10 целевых языков** — `zh-Hans` `zh-Hant` `ja` `ko` `ru` `hi` `es` `pt` `fr` `de`
- 🔌 **6 бэкендов с автооткатом** — OpenAI / Anthropic / DeepL / Azure / бесплатный Google / ваша подписка Claude
- 📁 **Переопределения на уровне проекта** — файл `.cc-translate.json` в репозитории переключает язык/режим (или отключает перевод) только для этого проекта
- 🔒 **Изоляция ключей** — API-ключи только в файле chmod-600; переменные окружения никогда не читаются
- 🛟 **Отказоустойчивость** — любая ошибка или таймаут откатывается к чистому английскому; сессия никогда не блокируется
- 🩺 **Встроенная диагностика** — `cctrans doctor` объясняет, почему ничего не переводится; `cctrans stats` показывает сэкономленные токены
- 🎚️ **Интерактивные настройки** — `cctrans settings` открывает односкринный редактор (стрелки); настройте отступы, маркер, модели и другое, базовое + расширенное

## 🚀 Быстрый старт

```bash
npm install -g cctrans@latest && cctrans install
```

Установка регистрирует хуки и открывает интерактивный редактор настроек (язык, режим отображения, бэкенд, API-ключ — `cctrans settings` открывает его повторно в любой момент). Затем **перезапустите Claude Code** — ответы станут двуязычными. Переключайте в любой момент, набрав `!cctrans off` / `!cctrans on` в поле ввода Claude Code (`!` — встроенный bash-режим CC: без вызова модели, без токенов).

**Уже установлено?** Обновляйтесь командой `npm install -g cctrans@latest` — изменения вступают в силу со следующего ответа (хук на каждом фрагменте запускается с диска заново); ваши настройки, ключи и зарегистрированные хуки не затрагиваются, повторная настройка не нужна.

## 🤔 Зачем это нужно

Две проблемы, одна архитектура:

**1. Claude Code упорно отвечает по-английски.** Skills и документация должны оставаться английскими, и даже с правилом «отвечай по-русски» в CLAUDE.md ответы сползают обратно в английский. Просить перевод заново — это целый ход модели и замусоренная история диалога.

**2. Работа на родном языке несёт скрытый «токенный налог» — именно на Claude.** Одно и то же значение стоит **в ~1.5–3 раза больше токенов**, чем по-английски (токенизатор Claude плохо сжимает нелатинские письменности), а 5-часовое окно и недельные лимиты Claude Code измеряются в токенах — неанглийские сессии сжигают тариф в 1.5–3 раза быстрее. Главное: **качество ответов — не проблема**: Claude набирает >90% в мультиязычных бенчмарках. Боль — чисто в стоимости.

| | японский | корейский | русский | хинди | китайский |
|---|---|---|---|---|---|
| Стоимость токенов vs английский | ~2–3× | ~2–3×+ | ~1.5× | ~2–3×+ | ~2–3× |

Issue Anthropic о лимитах с поправкой на язык ([#26401](https://github.com/anthropics/claude-code/issues/26401)) закрыт как *not planned* — официального решения нет.

**Поэтому самый дешёвый и правильный дизайн — ровно то, что делает этот инструмент:** сессия остаётся английской от начала до конца (промпты, транскрипт, контекст модели — ноль лишних токенов основного цикла), а ваш язык существует только там, где его читает человек: строка перевода только-для-отображения под каждой английской строкой, отрисованная отдельным дешёвым бэкендом.

Полные заметки исследования с источниками: [MOTIVATION.md](MOTIVATION.md).

## ⚙️ Как это работает

Использует нативный хук **`MessageDisplay`** Claude Code (v2.1.152+): он срабатывает при рендеринге каждого сообщения ассистента, передавая хуку завершённый фрагмент текста (`delta`); возвращаемый хуком `displayContent` **заменяет только отображение на экране**, не меняя сохранённое сообщение.

```
Claude стримит английский текст
        │  срабатывает на каждый фрагмент (stdin: turn_id/message_id/index/final/delta)
        ▼
  hook/message-display.js  ──►  src/interleave.js  ──►  src/translate.js
   (читает delta, проверяет     (проза / код / уже на целевом    (бэкенды + кэш)
    переключатель)               языке)
        │
        ▼  возвращает displayContent = "строка EN\n↳ строка перевода"
   Claude Code заменяет отображение на месте (оригинал остаётся в транскрипте/контексте)
```

> Проверено на CC 2.1.169: фрагменты `delta` **не перекрываются** (это не накапливающийся текст), обычный `\n` выводит два языка на отдельных строках, а блоки кода / пути / строки уже на целевом языке пропускаются автоматически.

## 🧩 Режимы отображения

`line` (по умолчанию) чередует строки: перевод под каждой английской строкой, в потоке вместе с ответом. `section` оставляет английский ровно таким, каким его стримит Claude, и вставляет **один сгруппированный перевод, когда блок завершён** — заметно спокойнее для ответов с большими списками. `message` идёт ещё дальше: весь ответ стримится на чистом английском, а **один сгруппированный перевод появляется в самом конце**:

```
Use these flags:
↳ Используйте эти флаги:

- Enable the cache
- Set a small timeout
- Prefer the batch API
  ↳ Включите кэш
  ↳ Задайте небольшой таймаут
  ↳ Предпочитайте batch API
```

```bash
cctrans mode section   # по блокам · cctrans mode message — весь ответ · cctrans mode line — вернуться к режиму по умолчанию
```

> В режиме section/message перевод появляется **когда блок (или весь ответ) завершён**, а не во время стриминга — с медленным бэкендом (например `claude-code`, 3–6 с/вызов) эта пауза заметна, поэтому здесь лучше всего подходят API-бэкенды. Если перевод блока не удался, английский не затрагивается — блок просто остаётся непереведённым.

**Под текстом или вместо него.** По умолчанию перевод показывается *под* английским (двуязычно). Хотите читать только на своём языке? `cctrans display replace` показывает перевод **вместо** каждой английской строки:

```bash
cctrans display replace   # только перевод · cctrans display append — вернуться к двуязычному режиму
```

Replace работает в **режиме line** (section/message по своей задумке сначала стримят английский, поэтому заменять нечего). Транскрипт и контекст модели в любом случае остаются на 100% английском; строка, которую не удалось перевести, сохраняет свой исходный текст, так что ничто никогда не пропадает.

**Хотите только свой язык, без английского вообще?** `cctrans only on` — это команда-ярлык ровно для этого (она задаёт режим line + replace). Английский даже не мелькает — Claude Code отрисовывает каждую строку уже переведённой — а любая строка, которую не удалось перевести, откатывается к английскому; блоки кода проходят без изменений. `cctrans only off` возвращает двуязычный режим.

## ❓ Диалоги с вопросами

Когда Claude Code просит вас выбрать из вариантов (интерактивный диалог-вопрос), сам вопрос, подписи вариантов и их описания тоже показываются на вашем языке — а ваш выбранный ответ по-прежнему доходит до модели на **английском**, поэтому её рассуждения остаются только на английском:

```
 ☐ 颜色偏好
Which color do you prefer?
↳ 您更喜欢哪种颜色？
❯ 1. Red
     ↳ 红色
     A bold, vibrant color
     ↳ 大胆、鲜艳的颜色
   2. Blue
     ↳ 蓝色
```

Это работает через хуки `PreToolUse`/`PostToolUse` в Claude Code (диалог-вопрос отрисовывается из входных данных инструмента, до которых оверлей сообщений не дотягивается). Он следует вашей настройке `display` — двуязычно в режиме append, только ваш язык в режиме replace. Включено по умолчанию; отключить можно командой `cctrans dialog off`. Если диалог не удаётся перевести вовремя, он показывается на английском без изменений.

> **Обновляетесь?** Запустите `cctrans install` один раз после обновления, чтобы зарегистрировать новые хуки диалогов. Восстановление ответа (сохранение ответа модели на английском) требует Claude Code ≥ 2.1.121 — `cctrans doctor` предупреждает на более старых версиях.

## 🌐 Бэкенды перевода

| Бэкенд | Требует | Скорость | Качество | Примечание |
|--------|---------|----------|----------|------------|
| `openai` (по умолчанию при наличии ключа) | `cctrans key openai` | ~1.4с/фрагмент | высокое | `gpt-4o-mini`, пакетный перевод строк, сохраняет код/пути |
| `anthropic` | `cctrans key anthropic` | ~1с/фрагмент | высокое | `claude-haiku-4-5` + structured outputs, строгие массивы строк равной длины (~$0.0005/фрагмент) |
| `deepl` | `cctrans key deepl` (бесплатно: 500 тыс. симв./мес) | ~0.5с/фрагмент | высокое | лучший классический MT; массивный API нативно выравнивает строки |
| `azure` | `cctrans key azure` (бесплатно: 2 млн симв./мес) | ~0.5с/фрагмент | среднее+ | опционально `cctrans key azure-region` |
| `google` | ничего | ~0.3с/фрагмент | среднее | бесплатный неофициальный эндпоинт; **резерв при отказе остальных** |
| `claude-code` | выполнен вход в `claude` CLI | ~3-6с/фрагмент | высокое | работает на вашей **подписке Claude** (`claude -p` headless) — без доплат, но заметно медленнее |

При сбое/таймауте основного бэкенда цепочка **автоматически откатывается на google** — сессия никогда не блокируется. Каждая переведённая строка кэшируется по хэшу «бэкенд + язык + содержимое».

API-ключи хранятся **только** в `~/.cc-translate/keys.json` (chmod 600) — задавайте их через `cctrans setup` / `cctrans key` или редактируйте файл напрямую. Переменные окружения оболочки никогда не читаются, поэтому ключи инструмента и ключи терминала не загрязняют друг друга.

Остальные настройки (бэкенд, язык, маркер, модели, эндпоинт Azure) находятся в `~/.cc-translate/state.json` — меняйте их командами `cctrans` или правьте файл напрямую.

## 🗣 Языки

```bash
cctrans lang zh-Hans  # упрощённый китайский (по умолчанию)    cctrans lang zh-Hant  # традиционный китайский
cctrans lang ja       # японский                               cctrans lang ko       # корейский
cctrans lang ru       # русский                                cctrans lang hi       # хинди
cctrans lang es       # испанский                              cctrans lang pt       # португальский
cctrans lang fr       # французский                            cctrans lang de       # немецкий
```

Для **CJK + русского + хинди** (нелатинские письменности) «строка уже на целевом языке» определяется бесплатно по диапазонам Unicode и пропускается. Для **испанского / португальского / французского / немецкого** (латинские письменности) вместо этого используется консервативная эвристика по стоп-словам — а если строка уже на целевом языке всё-таки уйдёт на повторный перевод, проверка на идентичность подавит эхо, так что в худшем случае это лишний вызов бэкенда, но никогда не неверная строка. Учтите, что для языков с латиницей экономия токенов меньше (~1.1–1.2× относительно английского против 1.5–3× для нелатинских — см. [MOTIVATION.md](MOTIVATION.md)); для них главная ценность — само двуязычное отображение.

Для китайского используются **коды письменности** BCP-47 (`zh-Hans`/`zh-Hant`) — традиционный китайский это письменность, а не регион; `zh-CN` / `zh-TW` принимаются как алиасы и нормализуются автоматически. Смена языка применяется мгновенно (хук перечитывает состояние при каждом вызове); кэши языков независимы.

## 📁 Переопределения на уровне проекта

Положите `.cc-translate.json` в корень репозитория (подойдёт любой родительский каталог рабочей директории), чтобы переопределить глобальные настройки только для этого проекта:

```json
{ "target": "ja", "mode": "section" }
```

или отключите оверлей для конкретного проекта через `{ "enabled": false }`. Переопределяемые поля: `enabled`, `target`, `mode`, `backend`, `marker`, `model`, `inputEn`, `inputMinChars`. Секреты не переопределяются — ключи остаются в `~/.cc-translate/keys.json`, а настройки эндпоинтов намеренно задаются только глобально. И `cctrans status` (запущенный внутри проекта), и `cctrans doctor` показывают, когда действует проектное переопределение. Относитесь к `.cc-translate.json` склонированного репозитория как к части его кода: он может, например, переключить бэкенд (в том числе на `claude-code`, который расходует вашу подписку) для работы в этом репозитории.

## 🩺 Диагностика проблем

Оверлей отказоустойчив по дизайну: любая ошибка деградирует до чистого английского, а не блокирует сессию — а значит, сбои происходят **тихо**. Когда ничего не переводится:

```bash
cctrans doctor
```

проверяет регистрацию хуков (включая устаревшие пути от старых установок), версию Claude Code (для MessageDisplay нужна ≥ 2.1.152), настроенный бэкенд и его ключ, живую связность (с замером задержки) и **последнюю ошибку хука** (хуки записывают её в `~/.cc-translate/last-error.json`, когда что-то ломается посреди стрима). А чтобы увидеть, что оверлей для вас уже сделал:

```bash
cctrans stats    # переведённые строки + оценка сэкономленных токенов основного цикла
cctrans cache    # размер кэша переводов; clear / gc для управления (лимит 200 МБ по умолчанию)
```

## ⌨️ Перевод ввода (beta)

`cctrans input on` включает хук `UserPromptSubmit`: когда промпт содержит достаточно нелатинского текста (по умолчанию от 4 символов — абсолютный счёт, поэтому пути и идентификаторы не размывают порог; настраивается через `cctrans input threshold <n>`), английский перевод прикрепляется как контекст, который модель считает каноничной инструкцией, и модель просят **отвечать на английском** — так двуязычный оверлей продолжает работать, а контекст разговора остаётся английским от начала до конца. (Проверено на CC 2.1.169: хуки не могут переписать сам промпт, поэтому оригинал остаётся в истории рядом с английским.) Английский ввод проходит без изменений; при любой ошибке промпт безопасно отправляется как есть.

> **Beta**: вызов перевода блокирует отправку каждого неанглийского промпта на ~0.5–1.5 с. По умолчанию выключено; мастер настройки спросит один раз. Отзывы → [issues](https://github.com/roy-jiang-opus/cctrans/issues)

## 📏 Поведение и ограничения (проверено)

- Хук срабатывает **во время стриминга** на каждый фрагмент; каждый фрагмент переводится и заменяется на месте — перевод появляется постепенно вместе с английским.
- Таймаут хука — **10 секунд**; внутренний гард инструмента — 9с. Любая ошибка / таймаут / превышение размера (>9000 символов) **безопасно откатывается к оригинальному английскому** — сессия никогда не зависает.
- Каждая переведённая строка **кэшируется** по хэшу содержимого (`~/.cc-translate/cache`, лимит 200 МБ применяется ежедневно); перерисовки и повторяющийся текст бесплатны. Все режимы используют общий кэш.
- В режиме section/message текст ещё не завершённого блока буферизуется в `~/.cc-translate/msgstate` (та же степень раскрытия данных на диске, что и у кэша); файл удаляется по завершении сообщения, а устаревшие файлы вычищаются через 24 ч.
- С `openai` каждый фрагмент — примерно один вызов API (~$0.0001) и ~1с задержки по сравнению с чистым английским; `google` быстрее, но качество чуть ниже.
- **Markdown-таблицы** остаются целыми: таблица проходит насквозь без изменений (поэтому сохраняется нативная отрисовка псевдографики Claude Code), а сразу после неё показывается переведённая копия таблицы — никаких больше строк, разорванных вставленным между ними переводом.
